# Rubik's solver

Визуализация и послойный солвер кубика 3×3×3.

Учебный проект на **C++20**: интерактивное окно с кубиком 3×3×3 (OpenGL), модель состояния «6 граней × 9 стикеров», повороты граней и средних слоёв, случайная перемешка при запуске и **автоматическое решение** методом, близким к классическому послойному (white cross / corners / second layer / OLL-like / PLL-like этапы в коде разбиты на `step1`–`step7`).

---

## Возможности

- Окно **800×800** (GLFW), отрисовка всех 54 стикеров с учётом «камеры» (поворот сцены стрелками).
- Повороты: **U, D, L, R, F, B** и обратные; дополнительно повороты **средних слоёв** (в коде — `rotateMiddle`, `rotateMiddleUp`, `rotateMiddleDown` и обратные).
- При старте выполняется **случайная перемешка** (~100 случайных ходов из допустимого набора).
- **Решение по Enter**: последовательность шагов выводится в консоль (`1)` … `7)`).
- Сохранение и загрузка состояния в файл **`cube.txt`** (работочая директория — там, откуда запускается исполняемый файл).
- Опциональный лог ходов в **`logs.txt`** (после нажатия клавиши записи лога — см. таблицу ниже).

Кодировка цветов в состоянии куба (индексы в `colors[6][9]`):

| Индекс | Цвет   |
|--------|--------|
| 0      | Синий  |
| 1      | Зелёный|
| 2      | Оранжевый |
| 3      | Красный |
| 4      | Жёлтый |
| 5      | Белый  |

## Структура репозитория

| Файл | Назначение |
|------|------------|

## Зависимости

- **CMake** ≥ 3.28  
- Компилятор с **C++20**
- **OpenGL**
- **GLEW**, **GLFW3**
- В `CMakeLists.txt` также подключены **glm** и **freeglut (GLUT)** через `find_package`; в исходниках проекта они **не используются**, но линковка может потребоваться для успешной сборки в вашей среде.

Удобный способ поставить библиотеки под Windows — [vcpkg](https://github.com/microsoft/vcpkg). Пример (из корня vcpkg, триplet подберите под свой компилятор):

```bash
vcpkg install glew:x64-windows glfw3:x64-windows glm:x64-windows freeglut:x64-windows
```

При сборке передайте toolchain-файл vcpkg CMake’у (как в [документации vcpkg](https://learn.microsoft.com/en-us/vcpkg/users/buildsystems/cmake-integration)).

## Сборка

CMake ищет библиотеки через `find_package`; абсолютные пути в репозитории не заданы. Нужно установить зависимости и указать CMake, где их искать — чаще всего через **toolchain vcpkg** или переменную окружения **`CMAKE_PREFIX_PATH`**.

**Пример с vcpkg** (путь к toolchain подставьте свой):

```bash
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=C:/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
```

**Linux и др.:** dev-пакеты OpenGL, GLEW, GLFW, GLM, freeglut из дистрибутива — сборка без toolchain, если `find_package` их находит.

**CLion:** *Settings → CMake → CMake options* — добавьте `-DCMAKE_TOOLCHAIN_FILE=.../vcpkg.cmake` или свой префикс, перезагрузите CMake-проект и соберите цель `rubiks_solver`.

Запуск (из каталога с `rubiks_solver.exe` / `rubiks_solver`, чтобы рядом находились `cube.txt` и `logs.txt` при сохранении):

```bash
./build/Release/rubiks_solver.exe   # Windows (Visual Studio / multi-config)
./build/rubiks_solver               # Windows Ninja / Linux / macOS (single-config)
```

## Управление

### Поворот сцены

| Клавиша | Действие |
|---------|----------|
| ↑ / ↓ | Наклон по X |
| ← / → | Поворот по Y |

### Ходы куба

| Клавиша | Ход |
|---------|-----|
| `U` | U |
| `I` | U′ |
| `D` | D |
| `S` | D′ |
| `L` | L |
| `K` | L′ |
| `R` | R |
| `E` | R′ |
| `F` | F |
| `G` | F′ |
| `B` | B |
| `V` | B′ |
| `2` | средний слой (вокруг вертикальной оси, «экватор») |
| `W` | обратный тот же средний слой |
| `5` | средний слой (верх/низ — см. `rotateMiddleUp` в коде) |
| `8` | средний слой (обратная операция к `5`) |

### Прочее

| Клавиша | Действие |
|---------|----------|
| **Enter** | Автоматическое решение (`solve()`) |
| **Tab** | Сохранить состояние в `cube.txt` |
| **Escape** | Загрузить состояние из `cube.txt` |
| **X** | Вывести состояние `colors` в консоль |
| **=** | Записать накопленные ходы в `logs.txt` |
